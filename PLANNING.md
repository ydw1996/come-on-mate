### 5. 사기진작비 (`/morale`)

#### 개요
- 직원 1인당 월 **5만원** 사기진작비 지급
- **2인 이상** 함께 사용 시 활용 가능
- 사용 후 양다윗 선임이 **박성배 수석님(경영지원)**께 이메일로 영수증 및 내역 보고

#### 사용 유형 4가지

| 유형 | 설명 | 금액 산정 방식 |
|------|------|--------------|
| **카페/아침** | 아침에 카페·베이글집 등 방문 | 각자 주문 금액 or n/1 |
| **점심 추가금** | 점심 2만원 한도 초과분 처리 | 초과금 n/1 or 누군가 더 부담 |
| **저녁 사기진작** | 저녁 식사로 팀 사기진작 | n/1, 잔여 예산 고려해 개인별 다르게, 일부만 사기진작비+나머지 개인카드 |
| **문화데이** | 매월 3째주, 남은 예산으로 자유 활동 | n/1 or 잔여 예산 고려해 배분 |

#### 금액 분배 방식
- **균등 분할(n/1)**: 총액 ÷ 인원
- **개인별 지정**: 각자 잔여 예산 상황에 따라 다른 금액 부담
- **대납**: 잔여 예산 많은 사람이 다른 사람 몫 대신 부담
- **혼합**: 일부는 사기진작비 차감, 초과분은 개인 카드

#### 메일 형식 (실제 발송 기준)
```
제목: [사기진작비] YYYY-MM-DD 사용의 건 (영수증 첨부)
       또는 [사기진작비] YYYY-MM-DD 사용의 건 (영수증 첨부 - 점심식사 추가금)

보내는 사람: 양다윗 선임 (didekdnit1996@commerceon.co.kr)
받는 사람: 박성배 수석님 (sbpark@commerceon.co.kr) — 경영지원
참조(CC): 함께 사용한 직원들

본문:
  안녕하세요 박성배 수석님
  양다윗 선임입니다.
  [유형] 이용으로 사기진작비 사용하여 메일 드립니다.
  날짜: YYYY년 MM월 DD일 요일

  1. 사용처: [장소명]
  2. 총 결제액: [금액]원 (총 N명)
  3. 사기진작비 차감 신청:

  (1) [이름] [직책]: [금액]원
  (2) [이름] [직책]: [금액]원
  ...

첨부: 영수증 사진 (카카오톡 사진)
```

#### 5-1. 내역 조회
- 구글 시트(`2026년 사기진작비`) 연동
- 내 사기진작비: 월별 사용금액 / 잔여금액 / 누적 합계
- 전체 조회: 전 직원 이번달 현황 테이블

**현재 상태:** 완료

#### 5-2. 영수증 메일 발송 (개발 중)

**플로우:**
1. 영수증 사진 업로드
2. GPT-4o Vision으로 장소명·날짜·총금액·메뉴 자동 인식 (Step1: 영수증 OCR)
3. 사용 유형 선택 (카페/아침 / 점심추가금 / 저녁 / 문화데이)
4. 장소명, 날짜 자동 입력 (수동 수정 가능)
5. 참여 인원별 이름(구글시트 직원 목록에서 선택) / 직책 / 차감금액 입력
6. ±100원 버튼으로 금액 조정
7. 메일 미리보기 (실제 발송 형식과 동일, PC는 오른쪽 sticky 레이아웃)
8. 발송 예정 → 박성배 수석님 수신, 참여자 CC

**구현 완료:**
- 영수증 업로드 + X버튼 삭제
- GPT-4o 2단계 분석 (영수증 → 메뉴별 참여자 자동 세팅)
- "샷추가" 등 옵션 항목 → 직전 메뉴에 통합
- 금액 100원 단위 step 입력 (총 결제액 포함)
- 이름 combobox (구글시트 직원 목록 드롭다운, 키보드 탐색/엔터 선택, IME 중첩 버그 수정)
- 이름 선택 시 직책 자동 입력 (`employee_profiles` 테이블 기반)
- 이름 선택 시 잔여 사기진작비 및 차감 후 잔여액 표시 (빨간색 경고 포함)
- 차감 합계 vs 결제액 차이 표시 (초과: 파란색, 부족: 빨간색)
- PC 2컬럼 레이아웃 (메일 미리보기 sticky)
- 영수증 Supabase Storage 저장 (서비스 롤 키로 RLS 우회)
- AI 스캔 애니메이션 (파란 웨이브)
- 영수증 라이트박스 (클릭 확대)
- **사용 유형별 금액 자동 계산 분리:**
  - 카페/아침: OCR 결과 기반 음료별 분리 (개별 금액 수동 조정)
  - 점심식사 추가금: 인원수 입력 → `(총액 - 인원×20,000) ÷ 인원` 자동 계산
  - 저녁식사 / 문화데이: 인원수 입력 → `총액 ÷ 인원` 자동 계산
  - 인원수 최대 10명 제한
- `employee_profiles` Supabase 테이블: 36명 직원 정보 (이름, 직책, 팀)
- 내역 조회: `employee_profiles`에 없는 직원 제외 필터링
- **OCR 자동 유형 감지:**
  - placeType 3가지: `cafe`(카페·베이커리) / `restaurant`(음식점) / `other`(문화·여가)
  - `cafe` → 카페 유형 자동 설정
  - `other` → 문화데이 유형 자동 설정
  - `restaurant` + 12~14시 → 점심식사 추가금 + 인원수 자동 추정 (총액 ÷ 20,000)
- **메일 템플릿 개선:** `3. 사기진작비 차감 신청:` 뒤에 차감 총액 표시
- **차감 합계 피드백:** 점심(초과분 기준) / 그 외(총 결제액 기준)으로 초과(파란) / 부족(빨간) 표시
- 인원 추가/삭제(X)가 headCount와 연동되어 금액 자동 재계산

**미결 사항:**

> ⚠️ 실제 메일 발송 기능 미구현 (Resend 연동 예정)
> ⚠️ 직원별 직책 관리 UI: 현재 Supabase Table Editor에서 직접 수정

- Storage: `morale` 버킷 (영수증 사진)

**현재 상태:** UI 완성 / 메일 실제 발송 기능 미구현
